# APP目录结构及基本介绍

在 `Cross` 框架中, 一个项目被拆分为多个app, 放在app目录下, 每个app都包含独立的控制器,视图及模板, 分别用于控制项目的具体部分, 一个常见的项目结构如下:

	├─app 应用模块目录    
	│  ├─admin 后台模块  
	│  │  ├─templates 模板文件夹
	│  │  │  └─web   
	│  │  ├─controller 控制器目录  
	│  │  └─view 视图文件目录 
	│  └─home 前台模块  
	│     ├─init.php 模块配置文件
	│     ├─templates 模板文件夹
	│     │  └─web  
	│     ├─controller 控制器目录  
	│     └─view 视图文件目录    
	├─lib 第三方类库 
	├─cache 缓存文件目录   
	├─config 配置文件目录  
	├─modules modules文件夹
	│   ├─admin
	│   └─common
	└─htdocs 可访问目录  
	  ├─admin  
	  └─home
